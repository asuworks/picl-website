Privacy Policy

Last updated: February 4, 2026

PICL ("the App") helps you create marketplace listings from photos and voice descriptions using AI. This policy explains what data is accessed, how it's used, and your choices.

Developer

Developed by Anton Suharev. Contact: suharev.works@gmail.com

No Developer Servers

The App has no backend. All cloud communication goes directly between your device and Google's services under your own account.

Google Services Used

  • Google Sign-In — OAuth 2.0 authentication.
  • Gemini API — analyzes photos and voice to generate listing text. No personal info is sent in prompts.
  • Drive API (drive.file scope) — uploads images/audio to your Drive. Cannot access other files.
  • Sheets API (spreadsheets scope) — exports listing data to your spreadsheet.
  • Google Fonts — downloads interface fonts.

All transmissions use HTTPS. Google's data use is governed by Google's Privacy Policy.

Local Storage

  • SQLite database — item records, image/voice references, listing data.
  • App files — captured photos and voice memos in private storage.
  • Preferences — location defaults, model selection, image quality.
  • Encrypted storage — API key via Android's encrypted keystore.

None of this data is transmitted to the developer.

Permissions

  • Camera — photograph items. Requested at point of use.
  • Microphone — record voice descriptions. Requested before first use.
  • Internet — communicate with Google APIs.

Data Sharing

No data is shared with any third party beyond the Google APIs listed above. No analytics, no ads, no crash reporting. The developer has no access to your data.

Retention & Deletion

  • Local data — deleted when you uninstall the App.
  • Drive/Sheets data — remains in your account until you delete it.
  • Account access — revoke at myaccount.google.com/permissions.
  • Gemini API data — subject to Gemini API Terms.

Children's Privacy

Not directed at children under 13. We do not knowingly collect data from children.

Security

HTTPS encryption on all connections. API key stored via Android's encrypted keystore. Cleartext traffic is blocked.

Changes

Updates will be posted here with a revised date. Continued use constitutes acceptance.
